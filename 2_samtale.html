1 Umiddelbare overvejelser omkring data?

Jeg har fet et dataset af film/serier, jeg bemrket at movie_id's har "tt" i ID'et, det giver mig en ide om at det nok er ID's der ikke er generet i datasettet, men formentligt kommer fra IMDB. Oftes er ID'er integers. Jeg bemrkede ligeledes, hvordan filmene er reduceret til 1 genre. P hjemmesiden IMDB kategoriseres strstedelen af filmene i flere genre. Dette kan pvirke resultatet i en model, eftersom man "reducerer" en film til n genre. Hvis man havde flere genre ville man kunne bruge MultiLabelBinarizer i sklearn til at "samle" en films genre til et array som kan bruges til at klassificere en film med flere genre effektivt.

4 Lav en klassifikationsmodel

Vores data skal renses for at kunne blive brugt effektivt i vores modeller. Til det har jeg brugt flgende pipeline:

  • cleantext: tal, linjeskift og datoer bliver udskiftet med tokens.
  • stop word removal: stopord som "er, et den, de, os, at, p, i" fjernes da de ikke giver mening at medtage i vores model og har ingen vrdi for vores model.
  • stemming: ord bliver "normaliseret", ord bliver reduceret til deres "rodform", enelser bliver fjernet osv.
  • tokenization: ord bliver lagt i en liste hvor elementer i listen er ord.

import re
import nltk
# nltk.download('punkt') ## skal kun kres en gang

from nltk.tokenize.regexp import RegexpTokenizer
from nltk.stem import PorterStemmer
from cleantext import clean

def clean_text(text):
  clean_text = re.sub(r'([A-Z][A-z]+.?) ([0-9]{1,2}?), ([0-9]{4})', '<DATE>', text)
  clean_text = clean(clean_text,
    lower=True,
    no_numbers=True, replace_with_number= r"<NUM>",
    no_currency_symbols=True, replace_with_currency_symbol="<CUR>",
    no_punct=True, replace_with_punct="",
    no_line_breaks=True 
  )
  return clean_text

def rmv_stopwords(tokens):
  stop_words = set(nltk.corpus.stopwords.words('english'))
  tokens = [word for word in tokens if word not in stop_words]
  return tokens

def stem_tokens(tokens):
  stemmer=PorterStemmer()
  Output=[stemmer.stem(word) for word in tokens]
  return Output

tokenizer = RegexpTokenizer(r'<[\w]+>|[\w]+')

Frst krer jeg 2 forskellige modeller jeg har en ide om kunne vre gode til den her slags opgave:

  • Naive Bayes model
  • Support vector machine (med linear kerne)

Naive bayes kan vre god til nr man har mindre mngder af data eftersom den kan hndtere et feature space i hje dimensioner uden at behve mange observationer (hvilket er en fordel nr vi ikke har meget data at kigge p). Den har ogs ofte mindre varians og der er derfor en mindre sandsynlighed for at modellen bliver overfittet.

Support vector machine(SVM) er en model som generelt er rigtig god til tekst klassificering, isr fordi den bruger regularization til at undg overfitting nr vi har f samples (film), men mange features (dimensionen i TFIDF vektor). Jeg ved ogs at SVM er brugt meget til tekst klassificerings opgaver som: spam dectection eller kategorisering af dokumenter.

Jeg overvejet ogs et nerual network, men de er ofte ikke s gode hvis man ikke har meget data, udover det s tager de ogs lang tid at trne og krver en del computerkraft.

    genre_df = df[df['genre'] == genre]

6 Hvad kan gre resultaterne bedre

Forslag 1: Brug crossvalidation og tune hyperparameters

Support vector machine modellen klaret sig bedre en naive Bayes modellen, men det var stadig ikke gode resultater. Resultaterne kan bla. forbedres ved at bruge cross validation og gridsearch til at finde de bedste parametre til vores TF-IDF vektorer og SVM. Her bruger jeg sklearn GridSearchCV som laver cross validation og finder de bedste parametre til vores model.

Forslag 2: Mere data

En anden mde vi kan forbedre vores model er ved at f mere data. Hvis vi har mere data kan vi forvente at se en forbedring i model resultater, men det kan ogs skabe problemer hvis den data man tilfjer ikke er god. Hvis der tilfjes en masse film i n genre fr vi potentielt en bias i vores model (det kan dog potentielt undgs ved at bruge oversampling), yderligere skal de film der tilfjes ogs have gode beskrivelser der er giver en tydelig indikation for filmens genre. Jeg opdaget at IMDB ikke tillader scraping uden en API key, som man skal kbe, der er open source alternativer, men de er meget begrnset i mngden af data man kan scrape per dag. Man kunne finde andre dataset med IMDB film, men jeg fandt det ikke passende at bruge nye dataset. import requests
# using the OMDB API to get movie titles from movie ids
def get_movie_title(movie_id, api_key):
    ## adding tt back to movie_id
    url = f'http://www.omdbapi.com/?i=tt{movie_id}&apikey={api_key}'
    response = requests.get(url)
    if response.status_code == 200: # OK
        data = response.json()
        if 'Title' in data:
            return data['Title']
        else:
            print(f"Error: {data['Error']}")
            return "NULL"
    else:
        print(f"Error: {response.status_code}")
        return "NULL"

def fetch_latest_movies():
    response = requests.get(IMDB_API_URL)
    response.raise_for_status()
    if response.status_code == 200:
        return response.json()
    else:
        return None

def add_new_movies(cursor, movies):
    for movie in movies:
        cursor.execute(
            "INSERT INTO movies (id, title, plot, genre, realese_date) VALUES (?, ?, ?, ?, ?)", 
            (movie['id'], movie['title'], movie['plot'], movie['genre'], movie['realese_date']))


def main():
    # Connect to the database
    conn = connect_to_db()
    cursor = conn.cursor()

    
    movies = fetch_latest_movies()
    if movies is None:
        print("No new movies found")
        conn.close()
        return
    
    add_new_movies(cursor, movies)

    conn.commit()
    conn.close()
